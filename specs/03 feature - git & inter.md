# 需求描述

1. 新增交互式功能

我们有了多个功能，使用起来会越来越复杂，提供一个可交互的操作方式将大大提升使用体验。

2. git 仓库选项
   经过加密的配置数据，可能是通过 git 的方式管理，提供方便的功能来更新、add & commit & push 更新

# 要求

1. 我们把实现分成两个阶段，先实现 git 仓库相关功能
2. 再实现交互功能
3. 注意多考虑下 corner case

- 比如配置+数据路径 是否是 git 仓库，且必须是 git 根路径，避免提交额外信息
- 更新（pull）的时候如果有冲突，那就不要强行 pull，让用户自己处理
- push 的时候如果有问题也提示

4. 要注意用户体验，说清楚问题原因
